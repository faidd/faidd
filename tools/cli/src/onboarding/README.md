# FAIDD CLI Onboarding Flow

The **Onboarding** layer handles the "First Launch" experience. It transforms a novice user into a protected "Sovereign."

## Responsibilities

- **Interactivity**: Management of questionnaires via `inquirer`.
- **Validation**: Environment verification (IDE, AI agent used).
- **Auto-Setup**: Automatic creation of the `.faidd/` bunker and the `.faiddrc.json` config if missing.

## "Elite" Vision

Onboarding is not just an installation; it's a security ritual. The UX must be fluid, informative, and provide an immediate impression of robustness.
